Part 1

Original Description

Create an XML version of your resume containing at least three sections: general information about you; your educational background, and your work experience. Create a schema for the XML document, and then create an XSLT for the XML document so that it can be rendered in a browser. Save all three files (resume.xml, resume.xsd, and resume.xsl) for Part 1 in the directory TMA1/part1.

Interpretation of the Part

Create an XML that contains information about my resume. Use XSD to define a schema in order to create a structure for my resume such as the types of information that should be contained in each categories of my resume. Finally, use XSLT to render the the info in my resume such that the XSLT caters to the schema definition and the code is reused for formats that are repeated.

Analysis and Design

The XSD schema is defined to create a general structure of the resume where the XML document is used to contain the actual content of the resume. Then, the XSLT is used to define the content of the HTML and convert the contents of the XML so that it can be rendered on the web. With the XSD schema, the general format of each tag and its corresponding restrictions are included to define what contents a resume should have. With the XSLT, it is created in a way such that a template is created for each unique tag. As some of the categories share common tags, these common tags can be reused by selecting the common tag and applying.

Documentation

The resume contains three categories, general info, educational background and work experience.

Within general info, name, position, phone number, email, address and website must exist. All tags except phone number must exist once. Phone number tag can occur a maximum of three times. Name must have first, middle and last tag where middle can occur or not. Phone number must have type and number tag where type can be from "home", "cell", and "work" and number is a string that is in the format of "xxx-xxx-xxxx". Email is a string where the it is filtered using the regex "[^@]+@[^\.]+\.[^\.]+" and address must contain number, street, apartment, city, province and postal code. Number is an int, street is string, apartment is int that is optional, province is string that must be a abbreviation of a Canadian province and postal code must be a Canadian postal code that is restricted using regex "[A-Z][0-9][A-Z] ?[0-9][A-Z][0-9]".

Within educational background, education tag must exist but can occur more than once. Within education, instituion, location, degree and period exists. Institution is a string, location must contain city and province and period must contain start and end. Start and end is a date type where it must contain month and year tag. Month must be an abbreviation and year must be between 1990 to 9999.

Within work experience, work tag must exist but can occur more tha once. Within work, name, location, position, period and description exists, Name is a string, position is string and description is composed of points that must occur once to five times which is also a string. Location and period is similar to the data type mentioned in educational background.

Interpretation of the Part

Create an application using AJAX technologies that includes 3 utility tools. The utility tools must be simple to use and user-friendly. Each tool must be event driven and produce results when enough inputs have been given. Updates to the document are made asynchronously.

Analysis and Design

A tool is displayed when the corresponding tool is selected using the navigation bar. Ajax call is made to retrieve the tool's html content and is appended to the document. After the tool's html has been loaded, its corresponding Javascript file is loaded in order to attach the event listener for its inputs.

For the unit converter, the user has the option to choose the type of units such as weight, length, area and volume. Then, the units available for each type of unit can be selected from the dropdowns of the two inputs. Users are able to input from any input boxes and the converted value will display from the other input box that is not being used. The converted value will be displayed whenever a user inputs a value or unit is changed. The conversion factors can be found in measurement_converter/conversion.js where for each type of units, a pair of units (unit1,unit2) are used as keys and the corresponding conversion factor is used as values. To convert from unit1 to unit2, value of unit1 must be multipled by the conversion factor. Conversly, to convert from unit2 to unit1, value of unit2 must be divided by the conversion factor.

For the mortgage calculator, the user can input the mortgage principal amount, interest rate, amortization period and payment frequency. The mortgage principal and interest rate are user inputs from an input box while the amortization period and payment frequency are dropdowns. When all the required values are included, mortgage payment, total payment and total interest payment is updated.

Although BMI is not a perfect indicator of health condition since it does not take into factor muscle mass, I chose to create the BMI calculator because it is still a good starting indicator of someone's health. The users are able to choose between metric and imperial units where metric units are in cm and kg and the imperial units are in ft/in and lbs. After height and weight are inputted, BMI is calculated and displayed asynchronously.

Documentation

utility_tools.js

  1. loadJS(tool): tool must be a string from ["measurement_converter", "mortgage_calculator", "bmi_calculator"]. Evaluates JavaScript code for the corresponding tool.
  2. loadHTML(tool): tool must be a string from ["measurement_converter", "mortgage_calculator", "bmi_calculator"]. Appends the corresponding HTML content of the tool to the main document.

measurement_converter.js

  1. convert(type, unit1, unit2, value): type is a string that specifies the type of unit. unit1 is a string that specifies the unit to be converted. unit2 is a string that specifies the unit to convert to. value is a numeric that is to be converted from unit1 to unit2.
  2. loadConverter(type): type is a string that specifies the type of unit. Loads inputs' event handlers with convert() function with correct units for proper conversion of units.

load_sidebar.js

  1. calcPayment(principal, interest, number): Calculate mortgage payment using the formula M = P*i*(1 + i)^n / ((1 + i)^n - 1) where M is mortgage, P is principal, i is interest rate per time frame and n is the number of times interest is accumulated.
  2. calcRegular(principal, yearlyInterest, frequency, years): Returns an array that contains payment per time frame, total payment and total interest paid. Payment per time frame is calculated using calcPayment, total payment is calculated by payment per time frame * frequency * years and total interest is calculated by total payment - principal. frequency is the number of time frame that occurs within a year and years is the amortization period.
  3. calcAccelerated(principal, yearlyInterest, frequency, years): Returns an array that contains the accelerated payment per time frame, total payment and total interest paid. Accelerated payment is calculated by monthly payment * 13 / frequency. Total payment is calculated using a while loop by summing up the payments until the amount that is required to pay equals 0. Total interest is total payment - principal.
  4. calcMortgage(principalValue, interestValue, amortizationValue, frequencyValue): Calculates mortgage information for the given user inputs and displays to the document.

bmi_calculator.js

  1. categorize(bmi): Categorizes bmi into underweight, normal weight, overweight and obesity.
  2. calcBmiMetric(cm, kg): Calculates BMI using units of cm and kg where bmi = kg / (cm / 100)^2.
  3. calcBmiImperial(ft, inches, lbs): Calculates BMI using units of ft, inches and lbs where bmi = 703*lbs / (ft*12 + inches)^2
